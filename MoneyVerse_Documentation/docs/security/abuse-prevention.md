# Abuse Prevention

## Risks

- Account-creation bots
- Credential stuffing
- PIN brute force
- Join-code enumeration
- Mission and reward farming
- Multiple-account referral abuse
- Offensive or identifying nicknames
- Parent or teacher misuse
- Suspicious exports
- Automated lesson completion
- Future payment fraud
- Future Plaid abuse

## Controls

Use layered limits by IP, account, session, device, route, and global thresholds. Add anomaly detection for impossible completion timing and replayed idempotency keys.

## Responses

- Progressive delays
- Temporary cooldown
- Session revocation
- Additional verification
- Guardian notification
- Human review
- Feature restriction
- Account suspension

Use neutral language. Do not automatically accuse a child of fraud.

## Appeals and review

Material restrictions require an auditable reason, review path, and false-positive handling.
